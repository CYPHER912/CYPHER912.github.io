# 实验报告

## 实验题目

利用云平台搭建个人博客

## 实验学时

2 学时

## 实验日期

2026 年 3 月 20 日

## 实验目的

本实验旨在通过实践操作，掌握基于云计算环境的网站部署技能。具体包括：构建环境并实现个人博客/网站的搭建，撰写实验报告，并将这份报告作为一篇博文发布在自己的博客/网站上。

## 实验环境

- 操作系统：macOS
- 开发工具：Terminal、Git
- 运行环境：Node.js、npm
- 博客框架：Hexo
- 博客主题：Butterfly
- 云平台/部署平台：GitHub Pages
- 自动化部署：GitHub Actions

## 实验内容

本次实验主要完成了个人博客的搭建与发布。首先，在本地配置 Node.js、Git 等基础环境，并初始化 Hexo 博客项目。随后，安装并配置 Butterfly 主题，对博客标题、作者、描述等基础信息进行修改，使博客具备基本的展示效果。

在博客功能搭建完成后，将项目源码上传至 GitHub 仓库，并结合 GitHub Actions 实现自动化部署。每次在本地完成文章编写并推送到仓库后，系统即可自动构建并发布到 GitHub Pages，实现博客内容的在线更新。最后，将本实验报告整理为一篇博客文章，发布到个人博客中。

## 实验步骤

1. 安装并配置 Node.js、Git 等运行环境。
2. 使用 Hexo 初始化博客项目。
3. 配置站点基本信息，如标题、作者、描述等。
4. 安装 Butterfly 主题并完成初步页面配置。
5. 编写测试文章，验证本地访问效果。
6. 将博客项目托管到 GitHub 仓库。
7. 配置 GitHub Actions，实现博客自动部署。
8. 通过 GitHub Pages 发布个人博客。
9. 将实验报告整理后以博文形式发布到博客中。

## 实验结果

通过本次实验，成功完成了个人博客的搭建与上线。博客能够在浏览器中正常访问，支持文章展示与后续内容更新。同时，借助 GitHub Actions 实现了自动化部署，提升了博客维护效率。实验报告也已作为博客文章发布，满足实验要求。

## 实验中遇到的问题及解决方法

在实验过程中，主要遇到了以下问题：

1. **Hexo 初始化目录非空**
   由于目标目录中已有文件，导致初始化失败。后续通过更换目录或清理目录后重新初始化解决。
2. **GitHub 仓库认证失败**
   在推送代码时，使用账号密码方式认证失败。后续改用 SSH 方式完成身份验证并成功推送。
3. **博客首页线上显示异常**
   本地访问正常，但线上首页空白。经检查后对文章内容与配置进行修正，并重新部署，最终恢复正常访问。

## 实验总结

通过本次实验，我掌握了利用云平台搭建个人博客的基本流程，了解了从本地环境配置、博客框架搭建到云端自动化部署的完整过程。同时，也加深了对 GitHub Pages 和 GitHub Actions 的理解，提高了实际操作能力。整体来看，本实验具有较强的实践性，对今后开展个人网站建设与维护具有积极意义。
